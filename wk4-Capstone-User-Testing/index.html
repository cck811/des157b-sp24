User Testing Results

Introduction

The purpose of the user testing was to evaluate the effectiveness of the interactive web project in helping users understand the impact of different economic conditions on wealth disparity.

The web project aimed to challenge the misconception that people are poor due to personal choices or laziness, by demonstrating how environmental and systemic factors can lead to different economic outcomes.

Testing Methodology

Moderated, in-person user testing sessions were conducted.

The target audience was students in a web design class, with varying levels of familiarity with the topic.

Two participants were involved in the testing.

The testing environment was a controlled classroom setting.

Tasks and Scenarios

  • Task 1: Navigate to the Simulator Page

    Task description: From the intro page, users were asked to navigate to the interactive simulator page.

    Expected outcome: Users should be able to find and click the "START" button to access the simulator.

  • Task 2: Interact with the Simulator

    Task description: Users were instructed to interact with the four sliders (Tax Rate, Minimum Wage, Social Welfare, and Market Forces) and observe the changes in the image section.

    Expected outcome: Users should be able to adjust the sliders and see the corresponding changes in the image, reflecting different economic outcomes based on the selected conditions.

  • Task 3: Leave a Comment

    Task description: Users were asked to leave a comment or thought in the comment section of the Simulator Page.

    Expected outcome: Users should be able to type and submit their comments without any issues.

Data Collection

Metrics measured: Task completion rates, time on task, user satisfaction ratings.

Observation techniques: Think-aloud protocol, note-taking by the moderator.

Questionnaires or surveys: Post-test feedback questionnaire.

Test Facilitation

Participants were given a brief overview of the project and instructed to complete the tasks while verbalizing their thoughts and actions.

No warm-up exercises were conducted.

The moderator's role was to guide participants through the tasks, answer questions, and take notes on their observations and feedback.

Analysis and Reporting

Qualitative analysis was performed on the observations and feedback collected during the testing sessions.

The reporting format will be a summary report highlighting key findings, user comments, and recommendations.

Recommendations and action items based on findings:

  • Improve the image transition when changing conditions in the simulator for a smoother experience.
  • Clarify the purpose of the comment section or consider moving it to a separate page.
  • Revisit the font and layout of the intro page for better accessibility and readability.
  • Consider adding a character or guide to make the design more relatable and "human".
  • Shorten the intro page content and make it more concise.
  • Enhance the visibility of the image description on the simulator page.
  • Add a "back" button to allow users to return to the intro page from the simulator page.

Logistics and Schedule

Timeline: Participant recruitment and testing sessions were conducted during a web design class period.

Resource allocation: One moderator, classroom setting, and participants' personal devices were used.

Contingency plans: In case of technical difficulties, participants would be provided with a backup device or the testing session would be rescheduled.
