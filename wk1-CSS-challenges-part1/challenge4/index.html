CSS Challenge 4

Fix The Paragraphs in the Sidebar

For this challenge, the paragraphs in the sidebar should be white, and using the Times New Roman typeface, but they are not. There is a rule on line 40 of the stylesheet that indicates these settings, but it is not working. Why not? What can you do to fix it, and what CSS feature is at the root of the problem? (Inheritance, cascade, or specificity).

Solution: The problem is specificity. Because the rule for the paragraphs in the article is prefaced with an ID for "content", those settings are more specific and overriding the settings for the paragraphs in the aside element. You can fix it by prefacing the aside p rule with #content. That way, the rule for the paragraphs in the aside element is more specific than the rule for the paragraphs in the article.

are other possbible solutions to this problem too, such as changing the #content p rule to article p, so that the specificty conflict is not created in the first place.
